# Project Summary

Explain the project from high level to concrete modules.

When summarizing:

1. Start with what the system does end-to-end.
2. Then describe the runtime path: entrypoint, config, command dispatch, agent loop, provider, tools, persistence.
3. Keep wording concise and easy to reuse in documentation or resumes.
4. Prefer concrete module names over vague architectural language.
